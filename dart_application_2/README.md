# Контент-платформа

## Предметная область

**Контент-платформа** — консольное приложение для управления контент-платформой, где пользователи публикуют статьи и подкасты, а также могут ставить лайки.

### Сущности

| Сущность | Описание | Связи |
|----------|----------|-------|
| **User** | Пользователь платформы | автор статей и подкастов; ставит лайки |
| **Article** | Статья | принадлежит User (authorId); получает лайки |
| **Podcast** | Подкаст | принадлежит User (authorId); получает лайки |
| **Like** | Лайк | от User (userId); к Article или Podcast (targetId + targetType) |

Связи:
- `Article.authorId` → `User.id` (FOREIGN KEY, CASCADE)
- `Podcast.authorId` → `User.id` (FOREIGN KEY, CASCADE)
- `Like.userId` → `User.id` (FOREIGN KEY, CASCADE)

## Что вынесено в каждый слой и почему

### `domain/` — Слой предметной области
Содержит модели данных и бизнес-правила (валидацию). Не зависит от способа хранения — знает только о структуре данных.

- **identity.dart** — абстрактный класс с `id`, общий предок всех сущностей.
- **user.dart, article.dart, podcast.dart, like.dart** — модели с полями, `fromMap()`/`toMap()`, `validate()`.
- **validation.dart** — переиспользуемые функции валидации, общие для всех моделей.

### `data/` — Слой данных
Отвечает за хранение и извлечение данных. Работает с SQLite через пакет `sqlite3`.

- **app_database.dart** — создаёт таблицы, реализует CRUD (insert, getAll, getById, getByAuthor/User/Target, delete). Принимает и возвращает объекты из `domain/`.

### `cli/` — Слой интерфейса
Отвечает за взаимодействие с пользователем через консоль.

- **menu.dart** — интерактивное меню с 16 пунктами: списки, добавление, удаление, фильтрация лайков, «показать всё из БД». Вызывает `validate()` перед сохранением и выводит ошибки.
- **input_helper.dart** — заготовка для будущего расширения ввода.

## Реализованные валидации

### 1. Обязательное текстовое поле (не пустое после trim())

Функция `requiredNonEmpty(value, fieldName)`:
- Проверяет, что строка не `null` и не состоит только из пробелов.
- Применяется во **всех сущностях** к полям `id`, `username`, `email`, `authorId`, `title`, `content`, `imageUrl`, `description`, `audioUrl`, `coverUrl`, `userId`, `targetId`.

### 2. Числовое поле больше 0

Функция `positiveInt(value, fieldName)`:
- Проверяет, что целое число > 0.
- Применяется в **Podcast** к полю `durationSeconds` (длительность подкаста не может быть ≤ 0).

### 3. Формат email

- В **User** поле `email` дополнительно проверяется на наличие символа `@`.

### 4. Формат даты/времени

Функция `validDateTime(value, fieldName)`:
- Проверяет, что строка корректно парсится в `DateTime`.
- Используется в меню при вводе даты (если потребуется ручной ввод).

### 5. Тип цели лайка

Функция `validLikeTargetType(value)`:
- Проверяет, что значение равно `article` или `podcast`.

## Перечень тестов (нагрузочное тестирование)

Файл `bin/benchmark.dart` — запускается командой `dart run bin/benchmark.dart`.

Тестирует 2 объёма: **100** и **200** записей.

| Операция | Что измеряется |
|----------|----------------|
| Вставка N пользователей | Скорость INSERT в таблицу users |
| Чтение всех пользователей | Скорость SELECT без фильтра |
| Чтение N пользователей по id | Скорость точечного SELECT по PK |
| Вставка N статей | Скорость INSERT с FOREIGN KEY |
| Чтение статей по автору | Скорость SELECT с фильтром |
| Вставка N подкастов | Скорость INSERT с FOREIGN KEY |
| Вставка N лайков | Скорость INSERT с FOREIGN KEY |
| Чтение лайков по пользователю | Скорость SELECT с фильтром |
| Чтение лайков по цели | Скорость SELECT с составным фильтром |
| Удаление N лайков | Скорость DELETE по PK |
| Удаление N подкастов | Скорость DELETE по PK |
| Удаление N статей | Скорость DELETE по PK |
| Удаление N пользователей | Скорость DELETE по PK (каскадно) |
